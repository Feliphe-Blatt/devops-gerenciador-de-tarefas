# Proposta Inicial - Gerenciador de Tarefas e Produtividade

## Nome do projeto
Gerenciador de Tarefas e Produtividade

## Tema da aplicação
Gerenciamento simples de tarefas pessoais ou acadêmicas, com foco em organização, acompanhamento e conclusão de atividades.

## Objetivo do sistema
Permitir que o usuário cadastre, acompanhe e conclua tarefas de forma centralizada, com uma interface simples e uma API organizada para suportar o ciclo completo do CRUD.

## Escopo em visão macro
O sistema exibirá uma lista de tarefas, permitirá criar novas tarefas, editar informações já cadastradas, alterar o status das atividades e remover itens quando necessário.

## Principais funcionalidades
- Cadastro de tarefas.
- Listagem de tarefas.
- Edição de tarefas.
- Alteração de status.
- Exclusão de tarefas.

## CRUD principal
O CRUD principal é o gerenciamento de tarefas.

## Tecnologias pretendidas
- Next.js com TypeScript no front-end.
- Spring Boot com Java no back-end.
- PostgreSQL como banco de dados.
- Docker e Docker Compose para execução e orquestração.
- Jenkins e SonarQube para automação e qualidade.

## Justificativa das tecnologias
- Next.js e TypeScript ajudam a construir uma interface moderna e consistente.
- Spring Boot acelera a criação de uma API REST clara e organizada.
- PostgreSQL atende bem a persistência relacional do CRUD.
- Docker e Docker Compose simplificam a reprodução do ambiente.
- Jenkins e SonarQube cobrem a automação e a análise de qualidade exigidas pela disciplina.
